Welcome to the Kash developer documentation. Build trading bots and agents against the protocol, embed markets in your website, and access real-time market data.

## Available APIs

<CardGroup cols={2}>
  <Card title="REST API" icon="bolt" href="/developer-docs/rest-api/overview">
    Programmatic access at `api.kash.bot`. Place trades, manage portfolios, receive HMAC-signed webhooks. API-key authenticated. The right surface for bots, agents, and market makers.
  </Card>
  <Card title="Public Embed API" icon="window" href="/developer-docs/public-embed-api/overview">
    Embed read-only markets in your website with iframes and a public REST surface at `app.kash.bot/api`. No authentication required. Designed for human/UI traffic.
  </Card>
</CardGroup>

<Note>
**Which one do I want?** If your code will be talking to Kash without a human in the loop — automated trading, portfolio analytics, market-making — you want the **REST API**. If you're embedding markets in a website or article for humans to look at, you want the **Public Embed API**.
</Note>

## What Can You Build?

### Trading Bots & Agents
Use the [REST API](/developer-docs/rest-api/overview) to place trades, monitor portfolios, and react to market events via webhooks. Supports idempotent retries, high-value confirmation gates, and per-key rate limiting.

### Market-Making Strategies
Combine the REST API for execution with the on-chain quote endpoint for live pricing. Per-key telemetry surfaces latency p50/p99 so you can monitor your own SLOs against ours.

### Embed Prediction Markets
Use the [Public Embed API](/developer-docs/public-embed-api/overview) to drop iframes and read-only market data into news sites, financial platforms, or community forums. Markets are read-only — users are directed to X to make predictions.

### Custom Dashboards
Pull real-time market data via either API depending on whether the dashboard is for humans (Embed API) or for an automated agent (REST API).
