# Notifications

<AccordionGroup>
  <Accordion title="What types of notifications does Kash send?">
    **Notification Categories**
    
    **Bets:**
    - **Bet Win:** Confirmation when your bet wins
    - **Bet Loss:** Notification when your bet loses
    
    **Leverage:**
    - **Position Confirmed:** When your leverage position is successfully opened
    - **Position Liquidated:** Alert when your leverage position is liquidated
    
    **Limit Orders:**
    - **Order Confirmed:** When your limit order is placed successfully
    - **Order Executed:** When your limit order is filled
    
    **Market Creation:**
    - **Market Accepted:** When your proposed market is accepted
    - **Market Rejected:** When your proposed market is rejected
    
    **Community:**
    - **Leaderboard Results:** Updates on leaderboard rankings
    - **Friend Tagged:** When friends tag you in discussions
    
    **Social:**
    - **Whale Activity:** Notifications when whales you follow make trades
    - **KOL Trades:** Updates when KOLs you follow trade
    - **Profitable Trader Activity:** Notifications from profitable traders you follow
    
    All notifications include links to the actual post and enable direct messaging with users.
  </Accordion>
  
  <Accordion title="Where can I receive notifications?">
    **Notification Delivery**
    
    **X Interface:**
    - **Direct Messages:** Notifications sent via @kash_bot DM
    - **Links to Posts:** Direct links to the actual X posts
    - **User Messaging:** Ability to DM users mentioned in notifications
    
    **Kash Web App:**
    - **In-App Notifications:** Notification center within the web application
    - **Links to Posts:** Direct links to the actual X posts
    - **User Messaging:** Ability to message users mentioned in notifications
    
    **Notification Categories Available:**
    - **All:** All notification types
    - **Bets:** Betting win/loss notifications
    - **Leverage:** Position confirmed and liquidated alerts
    - **Limit Orders:** Order confirmed and executed updates
    - **Market Creation:** Market accepted vs rejected updates
    - **Community:** Leaderboard results and friend tags
    - **Social:** Whale, KOL, and profitable trader activity
  </Accordion>
  
  <Accordion title="How do notifications work with posts and messaging?">
    **Notification Features**
    
    **Links to Posts:**
    - **Direct Access:** All notifications include links to the actual X post
    - **Context:** See the full conversation and market activity
    - **Engagement:** Interact directly with the post from the notification
    
    **User Messaging:**
    - **Direct Messaging:** Ability to DM users mentioned in notifications
    - **X Integration:** Seamless integration with X messaging
    - **Web App Messaging:** Message users through the Kash web application
    
    **Both Platforms:**
    - **X Interface:** Receive notifications via @kash_bot DMs
    - **Web App:** View notifications in the Kash web application
    - **Consistent Experience:** Same notification content across both platforms
    - **Real-time Updates:** Notifications delivered as events occur
  </Accordion>
</AccordionGroup>

<Warning>
Notifications contain links to actual posts and enable direct messaging. Always verify the authenticity of notifications and be cautious when clicking links from unknown sources.
</Warning>

<Tip>
Notifications are available in both the X interface via @kash_bot and the Kash web application, with consistent content across both platforms.
</Tip>
